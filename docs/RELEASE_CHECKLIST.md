# Release Checklist — Lila’s Path to Moksha

Практический чек-лист перед тем, как открыть игру бета-тестерам.
Проходим сверху вниз: каждый пункт — либо «ок», либо задача до релиза.

> Совет: копируй этот файл в issue/PR и отмечай галочками по мере проверки.

---

## 1. Безопасность

- [ ] `.env` не закоммичен (проверить `git ls-files | grep -E '(^|/)\.env$'` — пусто).
- [ ] `.env.example` содержит только плейсхолдеры, без реальных ключей.
- [ ] Telegram bot token используется только на сервере (`process.env.TELEGRAM_BOT_TOKEN`), никогда не импортируется в клиентские файлы и не читается через `import.meta.env.VITE_*`.
- [ ] Supabase service role key не появляется в клиентском бандле (`rg -n "SERVICE_ROLE" src` — только `.server.ts`).
- [ ] Ключ AI-провайдера (`LOVABLE_API_KEY`) используется только в server-функциях/route handlers.
- [ ] Telegram `initData` валидируется на сервере (HMAC от bot token) перед выдачей сессии.
- [ ] Endpoint Гуру требует авторизации и защищён от злоупотреблений (rate limit / usage table).
- [ ] Для всех приватных таблиц включён RLS, политики скоупятся по `auth.uid()`.
- [ ] Публичные роуты `/api/public/*` проверяют подпись/секрет вызывающего.

## 2. Геймплей

- [ ] Санкальпа: ввод, приватность, отображение в напоминаниях — работает.
- [ ] Бросок кубика: анимация, звук, значение 1–6, дебаунс.
- [ ] Ожидание рождения (вход по шестёрке) — работает.
- [ ] Перемещение фишки: пошаговая анимация корректна.
- [ ] Змеи опускают на правильные клетки, стрелы поднимают на правильные.
- [ ] Правило точного попадания на Мокшу (68) соблюдается — перелёт остаётся на месте.
- [ ] Рефлексии сохраняются в дневник (`journal_entries`).
- [ ] Дневник открывается, читается, записи не пропадают между сессиями.
- [ ] Пауза / продолжение / «Начать заново» ведут себя предсказуемо.
- [ ] Финальный итог (Мокша) показывается с ключевыми клетками и Санкальпой.

## 3. Telegram Mini App

- [ ] Игра открывается по кнопке из бота.
- [ ] `Telegram.WebApp.ready()` вызывается при монтировании.
- [ ] `expand()` вызывается там, где это уместно.
- [ ] `themeParams` подхватываются (тёмная/светлая тема выглядит корректно).
- [ ] BackButton закрывает открытый sheet/модалку, а не выходит из игры.
- [ ] Haptics не падают, если SDK недоступен (обёртка в try/catch).
- [ ] Браузерный fallback работает (dev-mode бейдж виден, игра играбельна вне Telegram).

## 4. Мобильный UX

- [ ] Нет горизонтального скролла на 320–430 px.
- [ ] Поле читабельно: номера, змеи (↓), стрелы (↑) видны без цвета.
- [ ] Нижняя панель действий (кубик / CTA) не перекрывается home indicator (safe-area).
- [ ] Экранная клавиатура не прячет кнопку «Сохранить» в рефлексии/фидбеке.
- [ ] Loading-состояния подписаны по-русски («Открываю путь…» и т.п.).
- [ ] Ошибки показываются человеческим текстом, есть способ повторить.

## 5. Приватность

- [ ] Текст «Поделиться» по умолчанию НЕ содержит Санкальпу и заметки — только по явному opt-in.
- [ ] Аналитика (`analytics_events`) не пишет содержимое Санкальпы, рефлексий и переписки с Гуру. `metadata` санитизируется.
- [ ] Контекст Гуру ограничен: текущая клетка, Санкальпа, последние ходы. Полный дневник не отправляется без явного согласия.
- [ ] Бот не пересылает приватные детали сессии в чат.
- [ ] Раздел «Приватность» в Настройках объясняет это по-русски.

## 6. Контент

- [ ] Все 72 клетки присутствуют (`src/content/*` + тесты целостности зелёные).
- [ ] Тон текстов клеток — тёплый, не осуждающий (tone-lint проходит).
- [ ] Правила / Помощь понятны новичку без пояснений.
- [ ] Онбординг короткий (4 экрана), пропускаемый, повторно открывается из Помощи.
- [ ] Русская копирайтинг-терминология согласована: **Путь, Санкальпа, Инсайт, Дневник, Гуру, Мокша**.

## 7. Тестирование

- [ ] Линтер проходит: `bun run lint`.
- [ ] Тесты зелёные: `bunx vitest run`.
- [ ] Продовая сборка собирается: `bun run build`.
- [ ] Пройден ручной сценарий в мобильном Telegram: вход → Санкальпа → бросок → змея → рефлексия → пауза → возобновление → Мокша.
- [ ] Пройден тот же сценарий в браузере (dev-mode / preview URL).

## 8. Запуск беты

- [ ] Форма обратной связи работает и пишет в `beta_feedback`.
- [ ] Известные ограничения задокументированы (в README или отдельном файле).
- [ ] README актуален: описание, стек, локальный запуск, ссылка на этот чек-лист.
- [ ] Скриншоты / GIF готовы (если планируется анонс).
- [ ] Ссылка на бота и на mini app проверена с чистого аккаунта.
